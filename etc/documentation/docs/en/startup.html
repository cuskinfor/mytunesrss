Starting MyTunesRSS

Mac OS X

The Mac OS X distribution contains an application bundle. Just double click the icon to start MyTunesRSS. You can drag the application to any place but usually it should be placed into the Applications folder.

Windows

The Windows distribution contains an executable which can simply be double clicked to start. You can place the MyTunesRSS folder anywhere but usually it should be put into the Programs folder.

Other systems

You can run MyTunesRSS on any java-enabled system. You simply need to open a command line, change the working directory to MyTunesRSS and start with the following command. 

java -jar MyTunesRSS.jar

Advanced Options

XML configuration

You can control the settings for MyTunesRSS with an XML file. Use the following command line option.

java -jar MyTunesRSS.jar -config <path_to_config_xml>

The configuration file has the following structure.

<?xml version="1.0" encoding="UTF-8"?>

<mytunesrss>
    <preferences load="true" save="true"/>
    <server port="8080" autostart="false"/>
    <datasource>
        <itunesxml>......</itunesxml>
    </datasource>
    <updatecheck>false</updatecheck>
    <database autoupdate="true"
              updateinterval="10"
              ignoretimestamps="false"/>
    <user name="default"
          password="PDp+yl6vs0Ft22LrTdBMYYw0a2I="/>
</mytunesrss>

The attributes "load" and "save" at the element "preferences" control the way the normal settings of MyTunesRSS, the one you make using the GUI, are treated. If you set "save" to "true", all settings are saved when MyTunesRSS is quit. If you specify "load" and set it to "true", the normal settings are loaded first and then the ones from the XML overwrite them. So you can have a basic configuration made through the GUI and modify a few values only using the XML file.

Most other options should be self-explainatory after reading the chapter about the configuration of MyTunesRSS. They are the same settings you see on the GUI. You can specify a user with a user name and password. Only the password is used while the user name has no meaning. Also if you specify more than one user, only the first one's password is used. The configuration uses the more flexible format since future versions of MyTunesRSS will support multiple users. The password is encrypted and Base64 encoded. You can generate such a password with the following call.

java -cp MyTunesRSS.jar de.codewave.mytunesrss.Crypt <password>

This encrypts and encodes the specified password and displays it for being copied into the XML file.

Starting without a GUI

If you want to use MyTunesRSS as a pure system service without any GUI, you can use the following command line option.

java -jar MyTunesRSS.jar -headless

For the server to start automatically and for MyTunesRSS to be configured correctly, you should first start it with the GUI once and go through the configuration. Or use the above mentioned XML based configuration. If MyTunesRSS is started without a GUI, you need a browser call to stop ist. Please note that for security reasons this call works only from the same machine that is running MyTunesRSS. So the browser needs to be started on the same computer. If you have a system without a graphical user interface, you need to use a text based browser like "lynx".

http://127.0.0.1:8080/shutdown
