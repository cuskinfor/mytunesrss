Die Konfiguration

Nach dem Start der Anwendung sehen Sie die Konfigurationsoberfläche. Die Einstellungen sind auf drei Seiten verteilt. Auf der ersten Seite befinden sich die allgemeinen Einstellungen.

Sie können den Port, auf dem der Server auf Verbindungen wartet, frei wählen. Der Standardport 8080 des Tomcat Servers sollte in den meisten Fällen problemlos funktionieren. Auf manchen Systemen kann die Benutzung von Portnummern unter 1024 eingeschränkt sein.

Normalerweise können Sie das Logging deaktiviert lassen. Wenn Sie eine Supportanfrage an Codewave Software stellen, schicken Sie bitte möglichst immer ein Log mit. Dazu schalten Sie es direkt nach dem Start von MyTunesRSS ein und benutzen die Anwendung bis zur Problemsituation. Mit dem Knopf "Log öffnen" können Sie sich das Serverlog anzeigen und kopieren.

Wenn Sie beim Starten des Servers die Meldung bekommen, dass Java nicht genügend Speicher zur Verfügung hatte, dann können Sie bei "Max. Speicher" den Wert erhöhen. Den neuen Wert speichern Sie mit dem Knopf direkt neben dem Eingabefeld. Nach dem Speichern des neues Wertes müssen Sie MyTunesRSS neu starten. Diese Option steht nur zur Verfügung, wenn Sie die spezielle Version für Windows oder Mac OS X benutzen.

Im unteren Teil der allgemeinen Einstellungen müssen Sie den Pfad zu der Datei "iTunes Music Library.xml" auf Ihrem System angeben. Sie können den Pfad direkt in das Eingabefeld schreiben oder mit dem (...) Knopf suchen und auswählen. Unter Windows liegt diese Datei normalerweise im Order "iTunes" unter "Eigene Musik". Auf Mac OS X sollte sie unter "iTunes" im Order "Musik" zu finden sein.

Auf der zweiten Seite finden Sie weitere Optionen. Sie können die Authentifizierung in der Webanwendung ein- und ausschalten und das entsprechende Kennwort vergeben. Das Kennwort wird unverschlüsselt gespeichert. Verwenden Sie daher bitte möglichst kein Kennwort, das sie an anderer Stelle für den Schutz von kritischen Daten verwenden.

Die Angabe eines Pseudo-Suffix kann nötig sein, wenn das Zielgerät, auf dem Sie den RSS Feed benutzen möchten, die Standardendung nicht unterstützt. Die Playstation Portable mit Firmware 2.60 kann AAC Dateien mit der Standardendung M4A nicht abspielen, sondern unterstützt nur die Endung MP4. Sie können also dem Zielgerät mit dieser Einstellung für die beiden unterstützten Typen MP3 und M4A eine alternative Endung anbieten.

Wenn das Zielgerät nur eine bestimmte Anzahl von Einträgen in einem RSS Feed unterstützt, dann können Sie in MyTunesRSS eine entsprechende Beschränkung konfigurieren. Die Playstation Portable beispielsweise zeigt nicht mehr als 100 Einträge pro Feed. Dadurch werden längere iTunes Wiedergabelisten automatisch in mehrere Listen aufgeteilt, so dass jede einzelne das Limit nicht überschreitet. Die Beschränkung greift auch bei manuellen Zusammenstellung eines Feeds.

Auf der dritten Seite finden Sie Informationen zum Entwicklerteam und die Bitte um eine Spende, falls das Programm Ihnen gefällt.

Unter den Konfigurationsseiten befinden sich die Knöpfe zum Starten und Stoppen des Servers und zum Beenden der Anwendung. Während der Server läuft, können Sie keine Einstellungen verändern. Wenn der Server gestartet wurde, wird in der Statuszeile die Adresse angezeigt, unter der der Server in Ihrem lokalen Netz erreichbar ist. Üblicherweise haben Sie eine einzeige lokale Adresse (z.B. 192.168.0.2). Wenn Sie mehrere Netzwerkschnittstellen haben (z.B. ein LAN und ein WLAN), dann haben Sie auch mehrere lokale Adressen. Es wird nur eine davon in der Statuszeile angezeigt. Wenn Sie alle lokalen Adressen sehen möchten, bewegen Sie bitte den Mauszeiger über den Statustext und schauen sich den Tooltip an.

Beim Beenden des Programmes wird ein eventuell noch laufender Server zuerst gestoppt. Die Fensterposition wird automatisch für den nächsten Start gespeichert. Wenn Sie Änderungen an den Einstellungen vorgenommen haben, werden Sie beim Beenden von MyTunesRSS gefragt, ob Sie diese für den nächsten Start speichern möchten.
