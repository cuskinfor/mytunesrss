MyTunesRSS starten

Mac OS X

In der Mac OS X Distribution befindet sich ein Applikationspaket. Sie starten MyTunesRSS einfach durch einen Doppelklick auf das Symbol. Sie können die Applikation in jeden beliebigen Ordner ziehen. Am besten legen Sie MyTunesRSS in den Programme-Ordner.

Windows

Unter Windows finden Sie im MyTunesRSS Verzeichnis die Startdatei MyTunesRSS.exe, die Sie durch einen Doppelklick starten. Sie können den MyTunesRSS Ordner an einer beliebigen Stelle ablegen. Am besten legen Sie ihn in das Programme-Verzeichnis.

Andere Systeme

Sie können MyTunesRSS auf jeden javafähigen System benutzen. Dazu müssen Sie in einer Kommandozeile in das MyTunesRSS Verzeichnis wechseln und dort das Programm mit dem folgenden Aufruf starten. 

java -jar MyTunesRSS.jar

Erweiterte Möglichkeiten

Konfiguration über eine XML-Datei

Sie können die Einstellungen von MyTunesRSS über eine XML-Datei beim Start der Anwendung angeben. Dazu starten Sie MyTunesRSS mit dem folgenden Parameter. 

java -jar MyTunesRSS.jar -config <path_to_config_xml>

Die Konfigurationsdatei hat den folgenden Aufbau. 

<?xml version="1.0" encoding="UTF-8"?>

<mytunesrss>
    <preferences load="true" save="true"/>
    <server port="8080" autostart="false"/>
    <datasource>
        <itunesxml>......</itunesxml>
    </datasource>
    <updatecheck>false</updatecheck>
    <database autoupdate="true"
              updateinterval="10"
              ignoretimestamps="false"/>
    <user name="default"
          password="PDp+yl6vs0Ft22LrTdBMYYw0a2I="/>
</mytunesrss>

Mit den Attributen "load" und "save" am Element "preferences" steuern Sie, ob die normalen Einstellungen von MyTunesRSS, also die über die Oberfläche gemachten, vor der XML-Datei geladen werden und ob die Einstellungen am Ende gespeichert werden sollen. Wenn "load" angeben und auf "true" setzen, dann werden zuerst die Einstellungen geladen und dann mit denen aus der XML überschrieben. Sie können also beispielsweise die XML auf einzelne Einträge beschränken, wenn Sie nur einige der Optionen überschreiben wollen.

Die meisten anderen Einstellungen sollten nach Lektüre des Kapitels zur Konfiguration von MyTunesRSS selbsterklärend sein. Es handelt sich jeweils um die gleichen Einstellungen, die auch über die Oberfläche vorgenommen werden können. Es kann ein Benutzer angegeben werden, wobei der Benutzername keine Rolle spielt und nur das Kennwort verwendet wird. Wenn Sie mehrere Benutzer angeben, wird nur das Kennwort des ersten Benutzers verwendet. Die XML nutzt das vorliegende Format, da folgende Versionen von MyTunesRSS mehrere Benutzer unterstützen werden. Das Kennwort ist verschlüsselt und Base64 kodiert. Sie können ein solches Kennwort mit dem folgenden Aufruf erzeugen.

java -cp MyTunesRSS.jar de.codewave.mytunesrss.Crypt <password>

Mit diesem Aufruf wird das angegebene Kennwort entsprechend verschlüsselt und kodiert ausgegeben und kann in die XML kopiert werden.

Start ohne Oberfläche

Wenn Sie MyTunesRSS als reinen Dienst ohne eine Oberfäche starten möchten, können Sie dieses mit dem folgenden Parameter tun.

java -jar MyTunesRSS.jar -headless

Damit der Server automatisch gestartet wird und MyTunesRSS korrekt konfiguriert ist, sollten Sie die Anwendung vorher einmal mit Oberfläche starten und entsprechend konfigurieren oder die oben gezeigte Möglichkeit einer Konfiguration mit Hilfe einer XML-Datei verwenden. Wenn Sie MyTunesRSS auf diese Weise starten, können Sie es mit Hilfe eines Webbrowsers mit folgendem Aufruf stoppen. Beachten Sie bitte, dass dieser Aufruf aus Sicherheitsgründen nur funktioniert, wenn der Browser auf demselben Rechner läuft wie MyTunesRSS. Auf Systemen ohne grafische Oberfläche müssen Sie entsprechend einen textbasierten Browser wie "lynx" verwenden.

http://127.0.0.1:8080/shutdown
